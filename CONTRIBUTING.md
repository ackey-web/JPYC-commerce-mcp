# コントリビューション ガイドライン

JPYC-commerce-mcp プロジェクトへの貢献を歓迎します。本プロジェクトは Apache License 2.0 の下で公開されています。

## はじめに

- 本リポジトリは現在 **Phase 0+（公開前強化フェーズ）** です。内部構造・DB スキーマ・ツール I/F は破壊的に変更される可能性があります。
- 大きな変更を提案する前に、必ず GitHub Issue を起票して議論してください。
- バグ報告・小さな改善は歓迎します。

## 開発環境

- Node.js **v20 以上**
- npm（または pnpm）
- Neon (PostgreSQL) アカウント
- Polygon Amoy testnet 用のウォレット（テスト用、少額で可）

セットアップ手順は `README.md` および `docs/setup.md` を参照してください。

## ブランチ・コミット規約

- ブランチ名は `feat/xxx` / `fix/xxx` / `chore/xxx` / `docs/xxx` / `test/xxx` のいずれか
- コミットメッセージは日本語・英語どちらも可。Conventional Commits 推奨
- 1 タスク 1 PR を基本とします
- 各コミットの末尾には、共同作業者の署名を含めても構いません（例：`Co-Authored-By: ...`）

## Pull Request フロー

1. Issue を起票し、担当者と合意
2. feature ブランチを作成
3. 実装・テスト追加
4. `npm test` がグリーンであることを確認
5. PR を作成（PR 本文にはテスト方法と影響範囲を記載）
6. レビュー → マージ

## コーディング規約

- ES Modules（`"type": "module"`）
- 2 スペースインデント
- `async/await` を優先
- エラーハンドリングは呼び出し元で適切に try/catch すること

## セキュリティ方針（必読）

本プロジェクトは **ノンカストディアル原則**で設計されています：

- MCP サーバーはユーザーの秘密鍵を一切保有しません
- すべての送金トランザクションは calldata を返すのみで、署名・送信は呼び出し元の責任です
- 例外は Merkle Root コミット用の運営ウォレットのみ（ユーザー資産とは完全に分離）

この原則を破る変更は受け付けられません。

## 禁止事項

- 秘密鍵、`.env` ファイル、接続文字列をリポジトリにコミットしない
- 本番 RPC URL・API キーを PR に含めない
- ライブラリに既存 GIFTERRA 系プロジェクトの Supabase テーブルへアクセスするコードを追加しない
- 法律・金融アドバイスと受け取られる文言をドキュメントに追加しない

## 法的・税務に関する注意

本プロジェクトは金融アドバイスを提供するものではありません。JPYC の法的扱い・税務は利用者自身が確認する必要があります。詳細は `docs/legal-notes.md` を参照してください。
